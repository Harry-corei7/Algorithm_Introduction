`难度：中等`

- 给定一个未排序的整数数组`nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
- 请你设计并实现时间复杂度为`O(n)`的算法解决此问题。

- 示例 1：
  - 输入：nums = [100,4,200,1,3,2]
  - 输出：4
  - 解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
- 示例 2：
  - 输入：nums = [0,3,7,2,5,8,4,6,0,1]
  - 输出：9
- 示例 3：
  - 输入：nums = [1,0,1,2]
  - 输出：3
---
`python语法：哈希集合HashSet起点法`
```python
class Solution(object):
    def longestConsecutive(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        # 重点概念：
        # (1) 哈希表-set, 字典-dict能在 O(1) 时间内判断某个元素是否存在。
        # (2) 列表-list, 元组-tuple只能在 O(n) 时间内判断某个元素是否存在。
        # 去重：把所有数字放入一个哈希集合（HashSet）
        num_set = set(nums)  
        max_len = 0
        for num in num_set:            # 遍历集合(O(n))
            if num - 1 not in num_set: # 起点法核心(找到一个起点)
                # 只有当 num-1 不在集合中，num 才是序列起点
                # 也就是说，实际存在几段序列，才会触发几次while循环
                # (只有一段序列的开头的数字，才触发一次计数循环)
                cur_num = num 
                cur_len = 1

                # 一直往后找连续数字
                while(cur_num + 1 in num_set):
                    cur_num += 1
                    cur_len += 1
                max_len = max(max_len, cur_len)
        return max_len
```

`python语法：区间合并字典法`
- 面试手撕不要用，太容易出错了
```python
class Solution(object):
    def longestConsecutive(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        length_dict = {}
        if(len(nums) == 0): return 0
        # 利用哈希表，使得nums元素在逻辑上相邻
        for i in range(len(nums)):
            pre = length_dict.get(nums[i] - 1, 0)
            suf = length_dict.get(nums[i] + 1, 0)

            if(nums[i] not in length_dict): # 只需要没有被录入到dict的数字
                length_dict[nums[i]] = (pre + suf + 1) # 核心，讲左右两个数的序列长度相加，并+1
                # 更新序列最两端的，端点的值
                length_dict[nums[i] - pre] = length_dict[nums[i]] # nums[i] - pre是pre所在序列的最左端
                length_dict[nums[i] + suf] = length_dict[nums[i]] # nums[i] + suf是suf所在序列的最右端
        result = max(length_dict.values())
        return result
```
