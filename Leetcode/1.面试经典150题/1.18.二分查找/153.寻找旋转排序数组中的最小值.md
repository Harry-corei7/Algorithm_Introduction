`难度：中等`

- 已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：
  - 若旋转 4 次，则可以得到`[4,5,6,7,0,1,2]`
  - 若旋转 7 次，则可以得到`[0,1,2,4,5,6,7]`
- 注意，数组`[a[0], a[1], a[2], ..., a[n-1]]`**旋转一次**的结果为数组`[a[n-1], a[0], a[1], a[2], ..., a[n-2]]`。
- 给你一个元素值**互不相同**的数组`nums`，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的**最小元素**。
- 你必须设计一个时间复杂度为`O(log n)`的算法解决此问题。

---
`python语法：个人理解版`
```python
class Solution(object):
    def findMin(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        left = 0
        right = len(nums) - 1
        # 特殊情况：只有一个元素
        if(left == right): return nums[left]
        # 思路: 相信一个条形图，每个条子的高度代表数值
            #   5
            # 4 *
            # * *     3
            # * *   2 *
            # * * 1 * *
            # * * * * *
        # (1) 经过了旋转：nums[left] > nums[right]
        # (2) 刚刚好旋转一轮：nums[left] < nums[right]，此时nums[left]是最小值
        while(nums[left] > nums[right]): 
            if(nums[left + 1] < nums[left]): # 当num[left+1]小于num[left]时，num[left+1]就是最小值
                left += 1
                break
            if(nums[right - 1] > nums[right]): # 当nums[right-1]大于nums[right]时，nums[right]就是最小值
                break
            left += 1
            right -= 1
        return min(nums[left], nums[right])
```
