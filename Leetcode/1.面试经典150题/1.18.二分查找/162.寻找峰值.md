`难度：中等`

- 峰值元素是指其值严格大于左右相邻值的元素。
- 给你一个整数数组`nums`，找到峰值元素并返回其索引。
- 数组可能包含多个峰值，在这种情况下，返回**任何一个峰值**所在位置即可。
- 你可以假设`nums[-1] = nums[n] = -∞`。
- 你必须实现时间复杂度为`O(log n)`的算法来解决此问题。

- 示例 1：
  - 输入：nums = [1,2,3,1]
  - 输出：2
  - 解释：3 是峰值元素，你的函数应该返回其索引 2。
- 示例 2：
  - 输入：nums = [1,2,1,3,5,6,4]
  - 输出：1 或 5 
  - 解释：你的函数可以返回索引 1，其峰值元素为 2；
     或者返回索引 5， 其峰值元素为 6。
---
- 这个算法的理论基础是峰值一定存在（因为数组两端是负无穷），并且通过比较mid与其相邻元素，可以确定峰值在哪一侧：
- 当`get(mid) < get(mid-1)`时：
  - 说明`mid`处于下降趋势
  - 左侧一定有峰值（因为左端是负无穷，一路下降到`mid-1`，所以中间必有上升转为下降的点）
  - 所以可以安全地放弃右半部分
- 当`get(mid) < get(mid+1)`时：
  - 说明`mid`处于上升趋势
  - 右侧一定有峰值（因为右端是负无穷，`mid`到右端必然要下降，中间必有峰值）
  - 所以可以安全地放弃左半部分

- 当mid是谷底时：
  - 第一个条件不满足
  - 第二个条件满足（因为 get(mid) < get(mid-1)）
  - 所以会执行 right = mid - 1，进入左半部分
- 为什么选择左半部分是对的？
  - 假设mid是谷底，比如数组 [5, 2, 4]，mid=2：
  - 左边是5 > 2
  - 右边是4 > 2
  - 根据算法规则：
  - 发现左边比当前大 → 向左搜索
- 为什么安全？因为从mid到左端，必然存在一个峰值
  - 证明思路：
    - 左端是负无穷
    - 走到mid-1时是5（比负无穷大）
    - 然后下降到mid（2）
    - 所以在左端到mid-1这段，必然有一个从上升到下降的转折点 → 峰值
---
`python语法-补充边界的二分查找`
```python
class Solution(object):
    def findPeakElement(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        left = 0
        right = len(nums) - 1
        # 辅助函数：输入下标i，返回nums[i]的值
        # 方便处理 nums[-1]以及nums[n]的边界情况
        def get(i):
            if(i == -1 or i == len(nums)):
                return float('-inf')
            return nums[i]
       
        # 二分查找-变式
        while(left <= right):
            mid = (left + right) // 2
            if(get(mid) > get(mid - 1) and get(mid) > get(mid + 1)): # 有-1，len(nums)两个边界，不用担心越界
                return mid
            elif(get(mid) < get(mid - 1)):
                right = mid - 1
            elif(get(mid) < get(mid + 1)):
                left = mid + 1
        return -1 
```
